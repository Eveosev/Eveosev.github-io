Regular Expression

Definition

Regular expression is a pattern-based method to find out specific symbols, strings or texts form objects. It can be ultilized to extract characters, replace object with a specific string, validate form, and so much more.

Text Normalization

Text normalization aims to convert text to a convenient and standard form. In this process, regular expressions play an import role.

Tokenization

The task of tokenization is separating out or tokenizing words from running text. In fact, tokenization is difficult to be applied. Why? Let’s see.

As is known to all, English words are often separated from each other by whitespace, but whitespace is not always sufficient. For examples, New York, rock ‘n’ roll, are sometimes treated as large words despite they contain spaces, while sometimes people would separate I’m into two words I and am. What’s more, emotions like :) or hashtags like #nlproc sometimes is supposed to be tokenized. Some languages like Chinese even do not have spaces between words. Hence, word tokenization becomes more difficult. [Jurafsky D. Speech & language processing[M]. Pearson Education India, 2000.]

Lemmatization

Lematization is the task of determining that two words have the same root or meaning, despite their surface differences. Essentially, lemmationzation is for processing morphologically (形态) comple languages.

Sentence segmentation

Sentences segementation is the task of breaking up a text into individual sentences, using cues (线索) like periods or exclamation (感叹) points.

Edit Distance

Definition

Edit distance is a metric that compares words and other strings. It measures how similar two strings are based on the number of edits it takes to change one string into other.

Minimum Edit Distance

Minimum edit distance between two strings is defined as the minimum number of editing operations (operations like insertion, deletion, substitution) needed to transform one string into another.

Levenshtein Distance

The Levenshtein distance between two sequences is the smalleset number of operations of converting one string to another. There are two version of Levenshtein distance.

Define $C$ is cost, hence

Levenshtein distance version 1

$$C_{deletion} = C_{insertion} = C_{substitution} = 1$$

Levenshtein distance version 2

Since any substitution can be represented by one insertion and one deletion. Therefore, we have

$$C_{deletion} = C_{insertion} = 1$$

$$C_{substitution} = 2$$
